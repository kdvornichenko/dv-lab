# Plan 09-03: Pet engine, overlay provider, and target discovery

<objective>
Render the pet inside the authenticated workspace shell with non-blocking overlay behavior, deterministic movement states, reduced-motion handling, privacy pose override, and explicit target discovery.
</objective>

<tasks>

<task type="auto">
  <name>Task 09-03-01: Implement non-React-frame pet engine and target discovery</name>
  <read_first>
    .planning/phases/09-implement-website-pet-widget/09-RESEARCH.md
    apps/web/lib/pet/pet-engine.ts
    apps/web/lib/pet/pet-engine.test.ts
    apps/web/lib/pet/pet-manifest.ts
  </read_first>
  <files>apps/web/lib/pet/pet-engine.ts, apps/web/lib/pet/pet-engine.test.ts, apps/web/lib/pet/pet-targets.ts</files>
  <action>
    Expand `apps/web/lib/pet/pet-engine.ts` with `createPetEngine(options)`, `step(deltaMs)`, `snapshot()`, `setPrivacyMode(enabled: boolean)`, `setReducedMotion(enabled: boolean)`, `setTargets(targets: PetTarget[])`, and `setCursor(position: { x: number; y: number } | null)`.

    The engine must clamp `deltaMs` to at most `64`, prioritize `privacy` over every other pose, prioritize `sleep` for reduced motion when privacy is false, and select jump/rest targets only from provided `PetTarget[]`.

    Implement concrete active behavior: idle/walk/run movement inside viewport bounds, activity-level tuning for `reduced`, `normal`, and `playful`, occasional target acquisition from explicit targets, a parabolic jump arc to the selected target, landing position clamped to the target/viewport, and a timed rest sequence on the target that can transition through `sit`, `lie`, and `sleep`. Cursor interest may affect facing or short curiosity movement, but must never allow jumping to unmarked cursor-adjacent elements.

    Create `apps/web/lib/pet/pet-targets.ts` exporting `PET_TARGET_SELECTOR = '[data-pet-target]'` and `listPetTargets()`. `listPetTargets()` must use `document.querySelectorAll<HTMLElement>(PET_TARGET_SELECTOR)` and `getBoundingClientRect()`, then reject rects narrower than `96`, shorter than `40`, or outside the viewport.

    Add `apps/web/lib/pet/pet-engine.test.ts` assertions for walking changing x-position over time, explicit-target jump start, jump arc y-position, landing clamp, rest-state transition on target, privacy override, reduced-motion sleeper override, and rejection of empty target lists for jump/rest behavior.
  </action>
  <verify>
    <automated>yarn --cwd apps/web test:pet</automated>
  </verify>
  <done>Pet engine supports concrete walk, explicit-target jump arc, landing, timed rest, privacy/reduced-motion overrides, target-only movement, and target discovery is explicit.</done>
</task>

    `WebsitePetProvider` must load persisted pet settings, run `requestAnimationFrame` only when enabled and active, cancel animation frame on cleanup, pause active movement when `document.hidden`, observe `html.privacy-mode`, and use reduced-motion detection to force sleeper behavior.
