  <name>Task 09-01-01: Add animated WebP asset manifest for pixel cat animations</name>
  <read_first>
    .planning/phases/09-implement-website-pet-widget/09-CONTEXT.md

  <files>apps/web/lib/pet/pet-manifest.ts, apps/web/public/assets/cat-widget/cat-sleep.webp, apps/web/public/assets/cat-widget/cat-walk.webp</files>
  <action>
    Before editing existing pet symbols, run GitNexus impact for `selectPetPose` and search current `catPetManifest` usages; report direct callers and risk. Create or update `apps/web/lib/pet/pet-manifest.ts` exporting `PET_POSES`, `PetPose`, `PetAnimationAsset`, `PetAssetManifest`, and `catPetManifest`.

    `PET_POSES` must contain exactly these pose strings: `idle`, `walk`, `run`, `jump`, `land`, `sit`, `lie`, `sleep`, `scratch`, `stretch`, `paw`, `fall`, `privacy`.

    Use the existing animated pixel WebP files under `apps/web/public/assets/cat-widget/`. `catPetManifest.id` must equal `cat`. `catPetManifest.animations.sleep.src` must equal `/assets/cat-widget/cat-sleep.webp`, width `208`, height `151`, and `animated: true`. `catPetManifest.animations.walk.src` must equal `/assets/cat-widget/cat-walk.webp`, width `370`, height `200`, and `animated: true`.

    Do not define sheet coordinates, frame IDs, `fps`, `backgroundPosition`, `frameWidth`, or manual frame indexes. Animated WebP owns the visual frame loop. Until additional animated art exists, every other required pose must still resolve to a deterministic animated WebP asset entry, but mark reused mappings with metadata such as `temporary: true` and `sourcePose` so later art replacement is explicit.
  </action>
  <verify>
    <automated>test -f apps/web/public/assets/cat-widget/cat-sleep.webp && test -f apps/web/public/assets/cat-widget/cat-walk.webp && rg -n "export const PET_POSES|catPetManifest|/assets/cat-widget/cat-sleep.webp|/assets/cat-widget/cat-walk.webp|animated: true|privacy" apps/web/lib/pet/pet-manifest.ts && ! rg -n "backgroundPosition|frameWidth|frames:" apps/web/lib/pet/pet-manifest.ts</automated>
  </verify>
  <done>`apps/web/lib/pet/pet-manifest.ts` exports the animated WebP asset contract and sleep/walk animated WebP files exist.</done>
