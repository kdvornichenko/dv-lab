# 11-01: Split memory and DB integration test scripts

## Goal

Developers and CI can distinguish memory-only smoke coverage from required DB-backed integration coverage.

## Tasks

- Add `test:memory`, `test:db`, and `test:db:optional` scripts to `apps/api`.
- Keep package `test` running memory plus optional DB integration for local compatibility.
- Make `db.integration.test.ts` fail when DB tests are required but no `TEACHER_CRM_TEST_DATABASE_URL` is set.
- Add root convenience scripts for API memory and DB integration.

## Verification

- `yarn --cwd apps/api test:memory`
- `yarn --cwd apps/api test:db:optional`
- `TEACHER_CRM_REQUIRE_DB_TESTS=1 yarn --cwd apps/api test:db` should fail locally if no test DB URL is set.
