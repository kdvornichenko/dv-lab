# 06-03 Plan: Sync Status, Retry, and Duplicate Prevention

## Objective

Make Calendar sync status reliable and visible, with manual retry and duplicate-prevention coverage.

## Tasks

1. Add sync status states for skipped, failed, pending, and synced records where needed.
2. Add retry endpoint/UI that reuses the existing sync record instead of creating duplicates.
3. Add contract tests for idempotent upsert and failed sync recovery.
4. Surface sync errors in the Calendar page and lesson detail surfaces.
5. Update Phase 6 summary and project state after verification.

## Success Criteria

- Retrying a failed lesson sync updates the same sync row.
- Repeated lesson save/sync calls do not create duplicate external event IDs.
- Calendar page shows connection and sync state clearly.

## Out Of Scope

- Background workers.
- Push notifications from Google Calendar.
