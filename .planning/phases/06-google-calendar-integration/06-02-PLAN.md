# 06-02 Plan: Calendar Selection and Lesson Event Mapping

## Objective

Use the stored Google token to list/select calendars and map lesson mutations to Google Calendar event payloads.

## Tasks

1. Add a Google Calendar API client module that uses stored encrypted tokens server-side.
2. Add calendar list endpoint and persist selected calendar in `calendar_connections`.
3. Define lesson-to-event mapping for title, time, duration, students, notes, and cancellation policy.
4. Upsert an event on lesson create/update/reschedule using the existing sync record.
5. Store external event/calendar IDs in `calendar_sync_events`.

## Success Criteria

- User can select a target calendar from the authenticated Google account.
- Creating/rescheduling a lesson creates or updates a single linked event.
- Calendar tokens never reach the browser.

## Out Of Scope

- Retry queue UI.
- Advanced recurring event rules.
