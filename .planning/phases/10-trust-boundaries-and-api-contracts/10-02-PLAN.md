# 10-02: Shared route parameter schemas and validation middleware

## Goal

Reject malformed dynamic route IDs before service calls using shared Zod schemas and existing typed validation error shape.

## Tasks

- Add shared route param schema exports in `packages/api-types`.
- Add an API `validateParam`/`validateParams` helper beside existing JSON/query validation.
- Apply validation to high-value dynamic route IDs in:
  - students;
  - lessons;
  - payments;
  - errors;
  - calendar blocks or calendar resource routes.
- Add route tests that malformed IDs return `400 VALIDATION_FAILED`.
