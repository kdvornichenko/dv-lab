# 10-01: Auth role hardening and forbidden response narrowing

## Goal

Remove user-controlled role trust and avoid returning sensitive authorization diagnostics to clients.

## Tasks

- Update `rolesFromUser` in `apps/api/src/middleware/auth.ts` to ignore `user_metadata.roles`.
- Keep `app_metadata.roles` and configured teacher-email fallback.
- Split permission-denied diagnostics into server log details and client-safe details.
- Add/update API tests for:
  - app metadata role accepted;
  - user metadata role ignored;
  - teacher email fallback accepted;
  - no trusted role receives 403;
  - 403 details omit user ID, email, roles, and permissions.
