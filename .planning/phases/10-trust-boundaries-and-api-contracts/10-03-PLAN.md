# 10-03: Schema-aware web API response parsing

## Goal

Make successful web API responses parseable with Zod schemas while preserving compatibility for broad existing `apiRequest` usage.

## Tasks

- Add an optional response schema parameter or options object to `apiRequest`.
- Preserve current generic-only behavior when no schema is supplied.
- Export and use response schemas from `packages/api-types` for core load endpoints first.
- Parse at least:
  - `/students`;
  - `/lessons`;
  - `/calendar/connection`;
  - one payment/settings/pet mutation path if schemas are already exported.
- Throw `TeacherCrmApiError` with a stable client-side code when a successful API response fails parsing.
- Add focused web tests if the current test setup supports API wrapper tests; otherwise add coverage through existing hook/model tests.

## Verification

- `yarn --cwd apps/web typecheck`
- `yarn --cwd apps/web test`
- `yarn typecheck`
